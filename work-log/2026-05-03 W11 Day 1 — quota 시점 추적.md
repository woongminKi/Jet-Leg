## 1. 비판적 재검토 — Option A 본격 vs lite

| 옵션 | 설계 | ROI |
|---|---|---|
| 본격 (~2h) | google-genai SDK 의 quota header 직접 파싱 | ❌ SDK 가 RPD 직접 노출 안 함 — 효과 제한 |
| HTTP layer 가로채기 | httpx interceptor 로 X-RateLimit-* 추출 | ❌ 복잡도↑ + SDK 변경 시 회귀 |
| **lite (~30분)** | **fast-fail 시점만 capture + 노출** | ✅ 채택 — `is_quota_exhausted` 재사용 |

→ lite 채택. SDK 한계로 정확 quota 노출 불가능하다는 점은 한계 #38 그대로 유지하되, 마지막 quota 소진 시점만 정확 capture 가능.
