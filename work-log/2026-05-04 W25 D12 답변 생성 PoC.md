## 2. 답변 생성 PoC ship — `/answer` 엔드포인트

### 2.1 변경

| 파일 | 변경 |
|---|---|
| `api/app/routers/answer.py` | 신규 — `/answer` 라우터 + `_gather_chunks` + `_build_messages` |

| `api/tests/test_answer_prompt.py` | 신규 — `_build_messages` 6 unit tests |

### 2.2 흐름

```
GET /answer?q=...&doc_id=...&top_k=5
  ↓
1) BGE-M3 embed_query (HF) — transient 실패 시 sparse-only fallback
2) supabase RPC search_hybrid_rrf (W25 D10 OR query 변환 + W25 D11 조사 strip)
3) doc_id 응용 layer 필터 (W11 Day 4)
4) top_k chunks 본문 + documents.title 조회
5) 검색 0건 → LLM 호출 회피 (quota 보호) + '자료에서 정보 찾지 못함' 답변
6) chunks → 한국어 prompt (faithfulness 보장: 검색 결과 외 추측 금지 / 인라인 [1] 인용)
7) GeminiLLMProvider.complete (3 retry + 백오프, 기존 인프라)
8) 응답: { answer, sources: [...chunk 메타...], query_parsed, took_ms, model }
```

### 2.3 설계 결정 (PoC 자율 결정)

| Q | 결정 | 이유 |
|---|---|---|
| Q1 endpoint | `/answer` 분리 | 검색만 사용 case 와 분리, quota 보호 |
| Q2 search 로직 재사용 | 직접 RPC 호출 (PoC minimal) | search router 600줄 메타·필터 재활용은 v1.5 통합 시점 |
| Q3 prompt | 한국어 + faithfulness | 사용자 페르소나 한국어 + 환각 방지 |
| Q4 출처 | sources[].chunk_id/doc_id/page/score | UI 직접 인용 가능 |
| Q5 model | Gemini 2.5 Flash | 기존 어댑터 재사용 (vision/tag/summarize 와 동일 인프라) |
| Q6 streaming | 동기 호출 | PoC — streaming 은 v1.5 |
| Q7 검색 0건 | LLM 호출 회피 + 명시 답변 | quota 보호 + 사용자 명확 |

## 3. Smoke 측정 (실 Gemini 호출)

### 3.1 G-S-001 — 검색 정확 → 답변 정확

```
q=소나타 전장 길이가 얼마나 돼?
sources: [chunk_idx 66, 85, 0, 87, 84]  ← 정답 [66, 85] top 2
answer: "소나타의 전장 길이는 4,910mm입니다 [1, 2]."
took_ms: 4723 (cold start 포함)
```

✅ 정답 청크 직접 retrieval + 정확 인용

### 3.2 G-S-005 — 검색 ranking 격차 → 답변 부분 부정확

```
q=소나타 트림 종류 뭐가 있어?
sources: [chunk_idx 65, 47, 46, 56, 22]  ← 정답 [39, 43] 누락
answer: "소나타 트림 종류는 스마트스트림 가솔린 2.0, ... [1]. 또한 N Line 트림도 ..."
```

⚠️ LLM 이 chunks 65 (엔진 정보) / 56 (N Line) 에서 트림 비슷한 정보 추출. 사용자 의도 (Premium / S / Exclusive / Inspiration) 와 다른 답변. **검색 ranking 격차의 직접 영향**.

### 3.3 G-S-009 — D-a-2 악화 case → 답변 부분 부정확

```
q=소나타 외장 색상 몇 가지야?
sources: [chunk_idx 45, 46, 96, 17, 0]  ← 정답 [44] 누락
answer: "제공된 자료에 따르면 소나타의 외장 색상으로는 녹턴 그레이 매트, 에어로 실버 매트, 얼티메이트 레드 메탈릭이 있습니다 [1]."
```

⚠️ chunk 45/46 의 부분 색상 정보만 들어옴. 사용자가 기대한 8종 (chunk 44) 누락. **D-a-2 ship 의 G-S-009 악화가 답변 품질에 직접 전이**.

### 3.4 자료 없는 질문 — Faithfulness 검증

```
q=쿠버네티스 설정은 어떻게 해?
sources: 5건 (검색은 됨, 무관 chunks)
answer: "제공된 자료에서 해당 정보를 찾지 못했습니다."
```

✅ Faithfulness 정상 — 검색 결과 있어도 관련성 없으면 LLM 이 명시적 거부

## 4. 본질 통찰 — 답변 품질 = 검색 품질

PoC 결과 **답변 품질이 검색 ranking 정확도에 직접 의존** 확인:
- 정답 청크 top-K 진입 → 정확 답변 (G-S-001)
- ranking 격차 → 부분 부정확 답변 (G-S-005/009)
- 무관 검색 → faithfulness 거부 (쿠버네티스)

→ **차수 B/C 의 사용자 경험 ROI 명확화**. mini-Ragas precision 0.05 개선이 추상 KPI 가 아니라 답변 품질 직접 영향. 단, SONATA 1건 over-fit 위험은 여전 → 사용자 자료 적재 후 본격 진입이 본질.
