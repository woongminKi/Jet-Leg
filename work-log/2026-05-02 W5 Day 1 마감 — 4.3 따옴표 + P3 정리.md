## 3. 핵심 변경 — 4.3 따옴표/괄호 보호

### 3.1 알고리즘

```python
if current and len(current) + len(sent) + 1 > _TARGET_SIZE:
    # W5 4.3 — current 가 odd-count 따옴표/괄호 이면 다음 sent 흡수
    if (
        _is_unbalanced_quote_or_paren(current)
        and len(current) + len(sent) + 1 <= _MAX_SIZE
    ):
        current = f"{current} {sent}".strip()
    else:
        pieces.append(current.strip())
        current = sent
```

### 3.2 검사 대상

- ASCII `"` 갯수 홀수
- 한국어 인용 따옴표 `"` `"` (여는 > 닫는)
- 한국어 인용 따옴표 `'` `'` (여는 > 닫는)
- 괄호 짝: `()` `[]` `{}` `「」` `『』` (여는 > 닫는)

### 3.3 안전성

- _MAX_SIZE 위반 위험 시 강제 cut (현재 동작 유지)
- 무한 흡수 방지 — `_MAX_SIZE` budget 검사로 자동 fallback

### 3.4 trade-off

- 인용문 중간 분리 방지 ↑
- 청크 길이 분산 ↑ (인용문 포함 청크는 최대 _MAX_SIZE 까지 늘어남)
- overlap 의 prefix 100자가 인용문 일부일 경우 dquote 짝이 깨질 수 있음 (회귀 영향 X)

## 5. flaky test 해소

### 5.1 원인

`test_concurrent_access_no_race_error` 가 worker 안에서 `patch.object(provider._client, 'post', ...)` 적용. 4 worker 가 thread pool 동시 실행 → patch context 가 thread 간 race 발생 가능. cold path (cache 비어있고 patch 미적용) 에서 실제 HTTP 호출 → dummy token 으로 401.

### 5.2 fix

patch 를 단일 with 블록으로 외부화 — 모든 worker 가 동일 mock 사용:

```python
with patch.object(provider._client, "post", return_value=...):
    with ThreadPoolExecutor(max_workers=4) as pool:
        futures = [pool.submit(worker, i) for i in range(4)]
        ...
```

### 5.3 검증

5회 연속 실행 모두 PASS (변경 전 1~2회 fail).
