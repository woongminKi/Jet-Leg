## 2. 옵션별 상세

### 옵션 A — Ragas 도입 (DoD ② ③ 회수)

**현 상태**: KPI 11개 중 6건 미측정 (Ragas 차단). golden v0.3 top-1 100% (20/20) 만 회귀 보호 중.

**사용자 액션 4단계** (`evals/README.md` §"W22 시점 사용자 액션 가이드" 참조):

1. **의존성 승인** (5분):
   ```bash
   cd api && uv add ragas datasets
   ```
   비용: ~10 MB + pandas/numpy. 외부 의존성 0 정책 변경 (W15~ 처음).

2. **평가 데이터셋 45건 누적** (수일~수주):
   - HWP/HWPX 10건 — 공공기관·법률·내규
   - PDF 10건 — 보고서·논문·매뉴얼
   - 스크린샷 15건 — 메신저·화이트보드·문서
   - DOCX 10건 — 브리핑·보고서
   - 각 자료당 3 QA 페어 (총 135 QA)
   - 현재 진척: 11/45 (24%)

3. **자율 sprint** (~1 W):
   - `evals/run_ragas.py` 신규 (5 metric: faithfulness / answer_relevancy / context_recall / context_precision / answer_correctness)
   - golden v0.4 (135 QA) → /search 호출 → ragas evaluate

4. **자율 sprint** (~30분):
   - `Makefile` `eval` target (DoD ③) — `make eval` 1회 재현

**회수 효과**:
- KPI 7건 측정 가능 (Faithfulness, Answer Relevancy, Context Recall, Context Precision, 환각률, HWP/PDF/Vision 성공률)
- DoD ② ③ 완료 → MVP 6항목 중 3 완료 (① ② ③)

SELECT count(*) FROM search_metrics_log;        -- 006

**회수 효과**: W15~W20 ship 자산 5개 즉시 활성. 자율 진행 결과 코드 가치 사용자 체감.
