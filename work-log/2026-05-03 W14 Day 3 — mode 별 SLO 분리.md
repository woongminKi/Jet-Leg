## 1. 비판적 재검토

### 1.1 옵션

| 옵션 | 설계 | 결정 |
|---|---|---|
| A | _rings: dict[mode, deque] 분리 | ❌ 합산 시 dict aggregate 추가 비용 |
| **B** | 단일 _ring + event["mode"] 키 + 응답 시 분리 | ✅ 채택 — 단순 + backward compat |

### 1.2 backward compat

- `record_search()` 의 `mode` 파라미터 default `"hybrid"` — 기존 호출 영향 0
- `get_search_slo()` 응답에 `by_mode` 신규 필드만 추가 — 기존 필드 보존
- 화이트리스트 외 값은 hybrid 강제 — 보수적 fallback

### 2.2 핵심 로직

```python
# search_metrics.py
_VALID_MODES = ("hybrid", "dense", "sparse")

def record_search(*, ..., mode: str = "hybrid") -> None:
    safe_mode = mode if mode in _VALID_MODES else "hybrid"
    event = {..., "mode": safe_mode}
    _ring.append(event)

def get_search_slo() -> dict:
    snapshot = list(_ring)
    overall = _compute_slo_for(snapshot)
    by_mode = {
        m: _compute_slo_for([e for e in snapshot if e.get("mode", "hybrid") == m])
        for m in _VALID_MODES
    }
    overall["by_mode"] = by_mode
    return overall
```
