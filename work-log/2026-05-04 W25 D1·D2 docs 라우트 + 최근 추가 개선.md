## 4. `<Link>` 중첩 회피 트릭 (new-arrivals-card)

행 전체 클릭 + 내부 태그 칩 클릭 동시 지원 시 nested anchor 는 hydration 경고 + 모호한 시맨틱.
해결:

```tsx
<li className="group relative ...">
  <Link href={`/doc/${doc.id}`} className="absolute inset-0 z-0 ..." />
  <div className="relative z-10 ...">
    {/* 제목 / 시간 */}
    <Link href={buildDocsUrl({ tag })}>...</Link>  {/* z-10 위에 떠 있음 */}
  </div>
</li>
```

- 행 Link 는 `absolute inset-0 z-0` — 시각 hover 영역만 차지, 클릭 위계는 z-10 본문 아래.
- 태그 Link 는 본문 (`z-10`) 안에 있어 행 Link 보다 클릭 우선순위가 높음.
- 결과: nested anchor 없음 (DOM 구조상 형제), 키보드 Tab 도 행 → 태그 순으로 자연스럽게 진입.

### 11.2 Fix 2 (P1 — UX 직접 영향) — new-arrivals-card 행 클릭 영역 확장

**원인**: `<div className="relative z-10 ...">` 본문이 행 Link (`absolute inset-0 z-0`) 를 stacking 위로 덮어 시간 표시·여백·`<p>` 영역의 클릭이 행 Link 에 도달 못 함. 사용자는 "행 어디든 누르면 doc 진입" 기대.

**해결안 — 옵션 (a) `pointer-events` 패턴 채택** (옵션 b 대비 stacking 모호성 0, 브라우저 호환성 검증 불필요):

```tsx
// 본문 div
<div className="pointer-events-none relative z-10 ...">
  {/* 자식들은 pointer-events 상속 — 클릭이 행 Link 로 통과 */}
  <Link className="pointer-events-auto ...">  {/* 태그만 별개 동작 */}
```

추가로 행 Link 에 `focus-visible:ring-offset-2 focus-visible:ring-offset-background` 추가 — 본문 z-10 으로 가려지지 않는 가시 focus ring 보장 (기존 `ring-2 ring-ring` 위에 offset 만 추가).

JSDoc 주석도 trick 설명을 신규 위계 패턴 (`pointer-events`) 으로 갱신.
