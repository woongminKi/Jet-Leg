## 2. W25 D14+1 sprint history (D1~D5)

### D1 — 측정 인프라 정정 + 자동 골든셋 + baseline

- `retrieval_metrics` graded relevance (acceptable_chunks 추가, weight 0.5)
- Recall@K 분모 cap K (nDCG IDCG 와 일관)
- `lru_cache cache_clear` 강제 (compare 토글 신뢰성)
- `auto_goldenset.py` 신규 — Gemini Flash-Lite 한국어 강제 prompt + acceptable BGE-M3 cosine 자동
- 43 query 자동 생성 (사용자 부담 0, ~$0.05)
- baseline: multi-doc top-1 0.9070, R@10 0.7138 — sonata 천장 효과 사라짐

### D1 오후 — NFC 정규화 (인제스트단)

- `chunk.py` / `doc_embed.py` 에 `unicodedata.normalize("NFC", ...)` 추가
- 효과 0 (11 docs 모두 이미 NFC) — 미래 안전망

### D2 — Query expansion (도메인 동의어 30 entry)

- `query_expansion.py` 신규 + search.py PGroonga query 통합
- multi-doc top-1 **+2.32pp**

### D3 — doc-level RRF 가산 ablation

- ENV `JETRAG_DOC_EMBEDDING_RRF_K` / `_WEIGHT` 노출
- **default k=10 / weight=2.0 sweet spot**
- multi-doc top-1 **+2.32pp** / MRR +1.55pp

### D4 — HyDE PoC

- `hyde.py` 신규 (Gemini 가상 문단 + LRU cache)
- multi-doc top-1 **+2.32pp**, latency +1~2s

### D5 — Reranker endpoint 변경 ⭐

- 이전: `sentence-similarity` pipeline = cross-encoder 미작동, -50% 회귀
- **신**: `text-classification` + `{q} [SEP] {p}` schema = 진짜 cross-encoder
- **multi-doc top-1 +4.65pp / MRR +3.10pp / doc-scope MRR +6.01pp** ⭐
- batch size 별 응답 형식 (flat / nested) robust 파싱
