## 1. 단계별 결과

### D1 오전 — 측정 인프라 정정 + 자동 골든셋 + baseline

**정정 항목**:
- `recall_at_k` / `mrr` / `ndcg_at_k` 에 `acceptable_chunks` 인자 추가 (graded relevance, weight 0.5)
- Recall@K 분모 cap K (nDCG IDCG 와 일관)
- `lru_cache cache_clear` 강제 (compare 토글 신뢰성)
- `chunk_recall_top3` → `chunk_recall_in_response` 명명 정정

**자동 골든셋** (`evals/auto_goldenset.py`):
- 11 docs × 4 chunks/doc → 43건 자동 생성
- Gemini 2.5 Flash-Lite 한국어 강제 prompt + `[SEP]`
- acceptable chunks BGE-M3 cosine ≥ 0.7 자동 라벨 (narrowness 차단)
- 사용자 부담 0, 비용 $0.05, 시간 ~10분

**baseline (43건)**:
| 측정 | 값 |
|---|---:|
| doc-scope chunk-level R@10 | 0.7138 |
| doc-scope MRR | 0.6519 |
| doc-scope nDCG@10 | 0.6798 |
| multi-doc doc-level top-1 hit | 0.9070 |
| multi-doc MRR | 0.9457 |

→ sonata 단일 (10건) 의 천장 효과 (top-1 100%) 사라짐 — **효과 측정 가능 baseline 확보**.

### D1 오후 — NFC 정규화 (인제스트단)

`chunk.py _to_chunk_records` + `doc_embed.py _pick_source` 에 `unicodedata.normalize("NFC", ...)` 추가.
**효과 = 0** — 기존 11 docs 모두 이미 NFC (HWP/HWPX 도) → 미래 인제스트 안전망 + 단위 테스트 +4건.

### D2 — Query expansion (도메인 동의어)

`api/app/services/query_expansion.py` 신규 — 30 entry 사전 (자동차 / IT / 의료 / 법률 / 학습).
opt-in `JETRAG_QUERY_EXPANSION` (default false). PGroonga query 만 expansion (dense path 무관).

**효과**:
| 측정 | OFF | ON | Δ |
|---|---:|---:|---:|
| multi-doc top-1 | 0.9070 | **0.9302** | **+2.32pp** |
| multi-doc MRR | 0.9457 | 0.9574 | +1.17pp |
| doc-scope R@10 | 0.7138 | 0.7074 | -0.64pp (noise) |

→ multi-doc 검색에서 외래어/약어 매칭 회복.

### D3 — doc-level embedding RRF 가산 + ablation

`JETRAG_DOC_EMBEDDING_RRF_K` / `_WEIGHT` ENV 추가. **default k=10, weight=2.0**.

**ablation 결과 (multi-doc, 43건)**:
| 설정 | top-1 | MRR |
|---|---:|---:|
| OFF | 0.9070 | 0.9457 |
| k=60, w=1 (이전 default) | 0.9070 | 0.9457 (영향 0) |
| k=10, w=1 | 0.9070 | 0.9496 |
| **k=10, w=2 (새 default)** | **0.9302** | **0.9612** |
| k=10, w=5 (과도) | 0.9070 | 0.9496 |

→ chunks RRF 와 균형 맞춘 가중치가 sweet spot. Δ top-1 **+2.32pp**, MRR +1.55pp.

### D4 — HyDE PoC

`api/app/services/hyde.py` 신규. Gemini → hypothetical document → (query + doc) embedding → dense path.
opt-in `JETRAG_HYDE_ENABLED` (default false). LRU cache 256.

**효과 (multi-doc, 43건)**:
- top-1 0.9070 → **0.9302** (+2.32pp)
- MRR 0.9457 → 0.9574 (+1.17pp)
- latency +1~2s (Gemini 호출, cache hit 시 0)

### D5 — Reranker endpoint PoC ⭐ **가장 큰 효과**

**curl 직접 검증으로 진짜 cross-encoder pipeline 발견**:
- 이전 (sentence-similarity): query-independent ranking → **-50% 회귀** (W25 D14+1 E)
- 신 (text-classification + `{q} [SEP] {p}` schema): query-dependent ranking 정상

검증 (3 query × 3 sentences):
| query | 시트 chunk | 안전 chunk | 날씨 chunk |
|---|---:|---:|---:|
| "시트 종류" | **0.9982** ✅ | 0.0182 | 0.0000 |
| "안전 기능" | 0.0001 | **0.9614** ✅ | 0.0001 |
| "날씨 어때" | 0.0001 | 0.0000 | **0.0316** ✅ |

→ query-dependent score 차별화 정상 작동.

**효과 (43건 multi-doc + doc-scope)**:
| 측정 | OFF | ON | Δ |
|---|---:|---:|---:|
| multi-doc top-1 hit | 0.9070 | **0.9535** | **+4.65pp** ⭐ |
| multi-doc MRR | 0.9457 | **0.9767** | **+3.10pp** |
| doc-scope R@10 | 0.7138 | 0.7286 | +1.48pp |
| doc-scope MRR | 0.6519 | **0.7120** | **+6.01pp** |
| doc-scope nDCG@10 | 0.6798 | 0.7100 | +3.02pp |

→ **모든 메트릭 향상**. 단 latency +2~3s/query (HF API).

## 3. 비판적 한계 (정직)

### 3.1 측정값 신뢰성
- N=43 통계 신뢰성 ±5~7pp (paper 급 ±2pp X)

- BGE-M3 cosine ≥ 0.7 acceptable threshold 휴리스틱 미검증

### 3.2 combined 효과 미측정
- 5 후보 모두 활성 시 효과 (additive vs interaction) 미측정
- 단독 효과의 산술합과 다를 수 있음

### 3.3 ENV default 결정
- 본 sprint default 변경 한 곳: `JETRAG_DOC_EMBEDDING_RRF_K=10` / `_WEIGHT=2.0` (코드 default, ENV off 인 한 적용 안 됨)
- 다른 4개 (expansion / hyde / reranker / doc-level RRF) 모두 default off — 사용자 결정 후 default on 변경 권고

### 3.4 reranker latency
- HF API cold start 시 +5~10s, warm 시 +2~3s
- 운영 활성화 시 사용자 인지 latency 우려
- LRU cache (4096 entries, key=(query, chunk_id)) 로 반복 query 0ms
