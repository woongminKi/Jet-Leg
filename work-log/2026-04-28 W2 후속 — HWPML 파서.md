## 1. 달성

### 1.1 매직바이트 게이트 — `_input_gate.validate_magic` 의 `.hwp` 분기 확장
- 기존: OLE2 prefix (`D0CF11E0A1B11AE1`) 만 허용 → 그 외 모두 400
- 신규: OLE2 ∨ HWPML XML 둘 다 허용. HWPML 검출은 `is_hwpml_bytes(head)` helper:
  - BOM(`EF BB BF`) 유무 무관
  - `<?xml` prefix
  - 첫 4096 byte 안에 `<HWPML` 루트 태그 substring 존재 (임의 XML 위장 차단)
- 거절 메시지: `"HWP 5.x(OLE2) 또는 HWPML(XML) 시그니처가 아닙니다."` (한국어 친화)

### 1.2 `HwpmlParser` 신규 (`app/adapters/impl/hwpml_parser.py`)
- 파싱: Python 표준 `xml.etree.ElementTree` — **의존성 추가 0**
- 본문 추출 로직:
  - `BODY > SECTION` 단위 순회 → 각 SECTION 안의 P 들
  - **leaf P 만** (자식 P 가 없는 P) — outer/inner 중복 회피
  - **`P/TEXT/CHAR` 직계 2단** 으로 한정 — PARAMETERSET·SECDEF 안의 스타일 메타데이터 CHAR 회피
- 메타데이터: `<HEAD/DOCSUMMARY>` 의 TITLE/SUBJECT/AUTHOR/DATE/KEYWORDS → `ExtractionResult.metadata.hwpml_*`
- ExtractionResult `source_type = "hwpml"` (DB CHECK 와 무관, 라벨링만)

### 1.3 extract dispatcher 분기 — `app/ingest/stages/extract.py`
- doc_type='hwp' 진입 시 raw bytes 의 첫 4KB 를 `is_hwpml_bytes()` 로 sniff
- 결과 True 면 `_hwp_parser` (Hwp5Parser) → `_hwpml_parser` (HwpmlParser) 로 교체
- DE-39 패턴(스캔 PDF rerouting = dispatcher 책임)과 일관

### 1.4 doc_type CHECK 제약 보존
- `documents.doc_type = 'hwp'` 그대로 사용 → 마이그레이션 추가 0
- 사용자 UI 라벨도 그대로 "HWP" 노출 (HWP 변형은 내부 구현 디테일)
