## 2. F1 — ChunksStatsCard (홈 추가)

### 2.1 변경 파일

| 파일 | 변경 |
|---|---|
| `web/src/lib/api/types.ts` | `ChunksStats` 인터페이스 + `Stats.chunks` 필드 추가 (W7 Day 3 백엔드 ship 후 미반영분 회수) |
| `web/src/components/jet-rag/cards/chunks-stats-card.tsx` | **신규** — SVG 도넛 + breakdown 진행 막대 |
| `web/src/components/jet-rag/home-grid.tsx` | PopularTags / MyDocStats 사이에 카드 추가 |

### 2.2 시각화 설계

- **도넛**: SVG `stroke-dasharray` 만으로 구현. 차트 라이브러리 (recharts/chart.js/d3) 의존성 추가 회피
  - `2π × r = c`, `effectiveLen = (effectivePct / 100) × c`, `dasharray = "${effectiveLen} ${c - effectiveLen}"`
  - 중앙 텍스트 `effectivePct%` (예: 59%)
- **breakdown**: 사유별 진행 막대 (`width: ${pct}%`), 한글 라벨 매핑
  - `table_noise → 표 노이즈`, `header_footer → 헤더·푸터`, `empty → 빈 청크`, `extreme_short → 초단편`

## 3. F2 — 검색 디버그 모드

### 3.1 변경 파일

| 파일 | 변경 |
|---|---|
| `web/src/components/jet-rag/result-card.tsx` | `debug?: boolean` prop + ChunkDebugPanel 컴포넌트 (chunk_id, chunk_idx, page, section_title, rrf_score, highlight_ranges, text_len, metadata 전체 키) |

| `web/src/app/search/page.tsx` | `searchParams.debug === "1"` 인식 → prop 전달 |

### 3.2 설계 결정

- **백엔드 search RPC 미변경** — `include_filtered` 같은 파라미터 추가는 단위 테스트 160/160 회귀 리스크. debug 본질은 "숨겨진 메타 펼쳐 보기" 이므로 프론트만으로 충분
- **URL 파라미터 `?debug=1`** — localStorage 대신 URL state. SSR/CSR 경계 처리 단순 + 공유 가능
- **모바일 숨김** — md+ 만 노출 (W6 Day 1 의 SearchSubheader queryParsed badge 정책과 일치)
