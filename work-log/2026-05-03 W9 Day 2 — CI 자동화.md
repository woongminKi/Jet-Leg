## 1. 비판적 재검토 (사용자 §1)

### 1.1 monitor_search_slo CI vs api/web CI

| 항목 | monitor_search_slo CI | **api/web CI** |
|---|---|---|
| 측정 대상 | 라이브 백엔드 검색 SLO | 단위 테스트 회귀 + tsc/lint |
| 의존성 | 백엔드 + Supabase + HF API | uv + pnpm 만 |
| secrets | 다수 필요 | **불필요** (mock 기반) |
| GitHub-hosted runner 동작 | 백엔드 미실행 → 의미 X | 즉시 실행 가능 |
| 가치 | 운영 환경 SLO 추적 (사용자 cron 적합) | **회귀 자동 탐지** (CI 본질) |

→ **api/web CI 채택**. monitor_search_slo 는 사용자 환경에서 cron 으로 실행하는 것이 자연스럽다는 점 README 에 명시.

## 2. 구현

### 2.1 `.github/workflows/ci.yml`

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  api-test:        # uv sync + python -m unittest discover tests (198 ran)
    runs-on: ubuntu-latest
    timeout-minutes: 10
    defaults:
      run:
        working-directory: api
    env:
      HF_API_TOKEN: dummy-test-token-ci
      GEMINI_API_KEY: dummy-test-token-ci
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
        with:
          python-version: "3.12"
          enable-cache: true
      - run: uv sync --frozen
      - run: uv run python -m unittest discover tests

  web-check:       # pnpm tsc --noEmit + pnpm lint
    runs-on: ubuntu-latest
    timeout-minutes: 10
    defaults:
      run:
        working-directory: web
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with: {version: 9}
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "pnpm"
          cache-dependency-path: web/pnpm-lock.yaml
      - run: pnpm install --frozen-lockfile
      - run: pnpm exec tsc --noEmit
      - run: pnpm lint
```
